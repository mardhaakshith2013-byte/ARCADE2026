## 📘 Lab Architecture Description
This lab systematically tracks, builds, and standardizes multi-tier infrastructure configurations inside Google Cloud using production-grade modular Terraform routines. The automated execution sequence tackles the following primary challenges:
1. **State Adoption:** Import pre-existing, unmanaged standalone Compute Engine instances (`tf-instance-1` and `tf-instance-2`) safely into the local configuration workspace state without disrupting uptime.
2. **Infrastructure Modularity:** Structural conversion of discrete infrastructure items into dedicated resource components (`modules/instances` and `modules/storage`).
3. **High-Availability Backend Migration:** Provisioning an isolated, secure Google Cloud Storage (GCS) cloud block asset to host the dynamic workspace locking mechanism, enabling seamless local-to-remote backend environment state migrations (`terraform/state`).
4. **Topology Optimization:** Vertical scaling of operational virtual instances from basic legacy system types (`n1-standard-1`) into highly optimized machine compute capacities (`e2-standard-2`).
5. **Lifecycle Management Validation:** Automating instance tracking destruction and dynamic replacement via granular node marking techniques (`terraform taint`).
6. **Advanced Custom Networking Enforcements:** Provisioning a secure Virtual Private Cloud (`tf-vpc`) containing segregated public/private networks across distinct geographic zones, establishing modular device associations, and configuring target firewall rule profiles (`tf-firewall`) to filter safe, inbound public web entry traffic.
